# Changelog

本文件记录 Eunet 项目在开发过程中的主要阶段性变更，按照实际 commit 历史进行整理，反映项目从底层网络能力构建到上层工具与界面逐步完善的全过程。本文档为过程性记录，而非单一版本说明。

---

## 2026-01-01 ～ 2026-01-02：底层网络与套接字基础阶段

### Added

* 新增 TCP / UDP 套接字的基础实现与对端信息获取接口
* 引入消息队列机制，用于统一管理 I/O 事件
* 增加对端主动关闭连接的错误语义
* 为字节缓冲区提供更灵活的检查接口

### Changed

* 套接字实现统一调整为阻塞模型
* 将对端获取逻辑上移至套接字基类
* 重构网络连接基类以适配新版套接字结构
* 移除旧的读写缓冲区依赖与 IO 协程上下文设计

### Fixed

* 修复消息队列中 fd 多次添加导致的边界异常
* 修复套接字缓冲区相关的潜在漏洞

### Test

* TCP / UDP 套接字相关单元测试逐步恢复并通过

---

## 2026-01-03：连接模型、错误体系与 HTTP 能力扩展阶段

### Added

* 完善统一错误模型与错误类型分类接口
* 将客户端实现对接至新版封装接口
* 引入 Boost 库并更新 HTTP 请求实现
* 实现更完整的 HTTP 数据结构与请求流程
* 增加事件类型到字符串的映射能力

### Changed

* 重构 HTTP 相关数据结构路径
* 明确 TCP 客户端的阻塞语义与等待时间（调整为 3000ms）
* 优化 TCP 客户端消息输出方式
* 共享连接之间的消息队列以减少重复逻辑
* 调整状态格式化接口返回类型

### Fixed

* 修复命令行在无参数情况下无法启动的问题
* 修复报错信息越界与对端关闭时的错误描述问题
* 修复传输结束后对端未正确关闭的异常行为

### Test

* 翻新 TCP / UDP 连接相关单元测试

### Docs

* 更新项目文档与构建教程说明

---

## 2026-01-03：TUI 界面引入与交互调整阶段

### Added

* 引入基于 FTXUI 的终端用户界面（TUI）
* 增加更详细的 UI 展示信息
* 实现事件时间线与状态变化的可视化呈现

### Changed

* 调整 TUI 结构，移除输入框以简化交互流程
* 美化页面布局，优化信息展示层级

---

## 2026-01-04：核心调度完善与稳定性修复阶段

### Added

* 允许请求终止后的事件继续进入统一调度流程
* 将事件快照中的错误信息调整为可选项
* 细化 Orchestrator 中的报错事件表达
* 更新生命周期状态机的状态定义与转移逻辑

### Changed

* 调整事件调度与状态推进顺序，保证时间线有序性

### Fixed

* 修复 TUI 代码中的逻辑错误
* 针对对端关闭场景补充更合理的错误处理路径

### Test

* 持续翻新并校验相关单元测试，确保行为稳定

---

以上变更均基于实际提交历史整理，用于反映 Eunet 项目在多次实现、重构与修复过程中的真实演进轨迹。
